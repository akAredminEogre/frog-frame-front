# DAILY SCRUM-01回目

## 本スクラムの作業予定
<!-- PLAN.mdの計画の中でどのユーザーストーリーに取り組むか記載してください。 -->

**Story-1: ユーザーとして右クリックメニューから「この要素を置換」を選択することにより、選択したテキストを含む要素全体を置換対象にできる**

1回目のスクラムでは、右クリックメニューの基本実装に取り組みます。以下のタスクを実施予定：

- [ ] background.tsで既存の「この部分を置換」メニューを削除
- [ ] 新しいコンテキストメニューアイテム「この要素を置換」を追加
- [ ] 新しいメニューアイテムのIDを`context-menu-replace-dom-element`として設定

## 修正予定のファイル
<!-- 修正予定のファイルを記載してください。 -->

- `host-frontend-root/frontend-src-root/entrypoints/background.ts`
  - 既存の「この部分を置換」メニューを削除
  - 新しいコンテキストメニューアイテム「この要素を置換」を追加
  - chrome.contextMenus.create()でメニュー項目を作成（ID: `context-menu-replace-dom-element`）
  - メニューアイテムのクリックハンドラーを実装

## 相談事項
<!-- 相談したいこと、質問したいこと、レビューしてほしいこと -->

- 既存の「この部分を置換」メニューの実装を参考にしながら、新機能との差別化をどのように図るか
 - →既存の「この部分を置換」は削除し、新しい「この要素を置換」を追加する形で実装してください。
- メニューの表示順序や文言について、UX的に適切な設計になっているか確認したい
  - → 文言はこれで問題ありません。また「この部分を置換」は削除し、「この要素を置換」を追加する形になるため、表示順序の考慮も不要です。
- 新しいメニューアイテムのIDとして`replace-element`で問題ないか
  - →既存のメソッド名との重複が気になります。右クリックの子メニューアイテムであることもわかるようなIDにすることをお勧めします。例えば`context-menu-replace-dom-element`など。
