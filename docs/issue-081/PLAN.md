# Issueの計画

Chrome拡張機能のリリース準備を行う。最低限の機能が完成しているため、以下の観点でリリースに必要な作業を実施する。

# DAILY-SCRUM単位のタスク
- ISSUE.mdを元に、開発タスクをデイリースクラム単位に分解する
- [x] リリース前のコード品質確認（lintエラー、未使用の依存関係、型チェック等）
- [x] マニフェストファイル（manifest.json）の最終確認（バージョン、権限、アイコン等）
- [x] README.mdの作成・更新（インストール方法、使い方、スクリーンショット等）
- [x] プライバシーポリシーの作成（Chrome Web Storeの要件）
- [x] ストア掲載用の説明文の作成
- [ ] テスト環境での動作確認
- [ ] Chrome Web Storeへの登録準備
