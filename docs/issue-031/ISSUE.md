# 概要
<!-- このチケットで解決したい課題 -->
実は、`https://agilemanifesto.org/iso/ja/manifesto.html` の
`<h1>アジャイルソフトウェア開発宣言</h1>` は、chromeの開発者ツールからDOMを確認すると
```
<h1>アジャイルソフトウェア開発宣言
</h1>
```
となっています。
そのため、`置換前` のフォームに、：``<h1>アジャイルソフトウェア開発宣言</h1>`と入力すると、
`<h1>アジャイルソフトウェア開発宣言</h1>` と一致せず、置換ができません。

なので、タグの開始前に改行コードや空白があっても、ユーザー側としては意識しなくても置換ができるようにしたいです。

想定するユースケースとe2eテストの内容
- ブラウザで、`https://agilemanifesto.org/iso/ja/manifesto.html` を開く
  - `<h1>アジャイルソフトウェア開発宣言</h1>` と一致するDOMがあることを確認
- ポップアップを開く
- `URLパターン (前方一致):` フォームに `https://agilemanifesto.org` が自動入力されていることを確認する(このe2eロジックは実装済み)
- ポップアップに次の値を入力
  - 置換前：`<h1>アジャイルソフトウェア開発宣言</h1>`
  - 置換後：`<h2>アジャイルソフトウェア開発宣言</h2>`
  - `正規表現を使う` のチェックは空にしておく
- 保存ボタンを押す
  - モーダルが開き `保存して適用しました！` と表示されることを確認
  - ブラウザで表示されているページの `<h1>アジャイルソフトウェア開発宣言</h1>` が `<h2>アジャイルソフトウェア開発宣言</h2>` に置換されていることを確認
