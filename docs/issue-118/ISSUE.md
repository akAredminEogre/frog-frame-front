# 概要
<!-- このチケットで解決したい課題 -->
いつかのissueの議論で出た、issueに対して複数のブランチを紐づける運用について議論をしたい

## 実現したいこと
- issueに対して複数のブランチを紐づける運用ルールを決める
  - メインのissueのブランチ名はこれまでと同じ
  - それにサブブランチを作る
    - issue-119-daily-scrum-01 のような命名規則
- issue全体の開発
  - これまで
    - daily-scrum1個の実装レビューが終わってから次のスクラムがスタートしていた
  - 今後
    - PLAN.mdの時点でデイリースクラムでの実装予定を全て決めておく。
    - 開発者がclaude codeに開発指示を出したら、claude codeは、daily-scrum毎にサブブランチを作成して、コーディングをし、メインブランチにPRを出す。その後、そのコミットから次のdaily-scrumのサブブランチを作成して、同様にコーディングをし、前のブランチに対してPRを出す。(後述)

### フロー(ざっくり)
- issue開始時
  - /workflow-create-branchでメインブランチを作成
  - ISSUE.mdを作る
    - 開発者が記入
  - PLAN.mdを作る
    - 既存の /workflow-create-plan-with-issue で実行
    - 現在相談事項がなければ次のステップに進むフローになっているが、それはやめる
    - この段階で、daily-scrum毎の実装予定を全てPLAN.mdに記載しておく
    - ここで開発者がレビューをし、承認したら1個目のdaily-scrumを開始する
- daily-scrum開始時
  - claude codeは、規約や、
