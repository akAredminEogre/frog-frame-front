# 概要
正規表現を使用した置換でも、HTMLに含まれる改行コードを無視して置換が実行されるようにする機能の実装。

現在の実装では通常の文字列置換では改行コードを無視する機能があるが、正規表現を使用した置換では改行コードが考慮されないため、実際のHTMLの改行が含まれるコンテンツに対して期待通りの置換ができない問題がある。

## 関連リンク
- `ignore-crlf-replace-with-regex.spec.ts` - 本機能のE2Eテスト

# 受け入れ条件
- 正規表現を使用した置換でも改行コードを無視して置換が実行される
- `ignore-crlf-replace-with-regex.spec.ts`のE2Eテストが正常にパスする
- 既存の通常文字列置換機能に影響を与えない

# 心配事
- 正規表現パターンの変換処理が複雑になる可能性
- 既存の正規表現処理への影響
- テストファイル自体の問題（構文エラーやチェックボックス処理）

# 制限事項
- 既存の`createRedundantPattern`メソッドの処理パターンを踏襲する
- 正規表現の特殊文字処理に注意が必要
- E2Eテストの実行環境に依存
