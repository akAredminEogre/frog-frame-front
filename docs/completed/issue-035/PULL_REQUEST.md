# ISSUE-035 feat: 正規表現置換で改行コードを無視する機能を追加

## 概要と理由
本変更は、正規表現を使用した置換処理において、HTMLソース内の改行コード（CRLF）を無視してパターンマッチングを行う機能を追加するものです。

従来の実装では、通常の文字列置換では改行コードを無視できていましたが、正規表現置換では対応していませんでした。これにより、実際のHTMLコンテンツに対して意図した通りの置換ができない問題がありました。この修正により、正規表現の柔軟性を維持しつつ、より正確な置換を実現します。

## 主な変更点
- **改行コードを無視する正規表現パターンの生成ロジック追加**
  - `HtmlContent.ts`内の`createRedundantPattern`メソッドを拡張し、正規表現パターン内の`<`と`>`の間に`\s*`を挿入することで、タグ間の改行や空白を無視できるようにしました。
- **メソッドの統合によるリファクタリング**
  - 従来、通常文字列用と正規表現用に分かれていたパターン生成メソッド（`createRedundantPattern`と`createRedundantRegexPattern`）を一つに統合し、コードの重複を排除しました（DRY原則）。
  - TypeScriptの型安全性を向上させるため、`isRegex`フラグの`undefined`をハンドリングする処理を追加しました。
- **E2Eテストの修正**
  - `ignore-crlf-replace-with-regex.spec.ts`内の構文エラーを修正し、正規表現置換を有効にするためのチェックボックス操作を正しく行うようにしました。

## 補足
- **無限ループの懸念について**
  - 正規表現による`replace`処理は、一度の実行で全てのマッチ箇所を置換するため、文字列置換で懸念された無限ループのリスクはないことを確認済みです。
- **コード品質の向上**
  - 今回のリファクタリングにより、コードの保守性と可読性が向上しました。
