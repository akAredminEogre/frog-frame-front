## スクラム01 の振り返り

### Keep
- プライバシー情報を発見後、速やかに削除作業を開始できた
- レビューコメントに対して迅速に対応し、段階的に問題を解決できた
- git rm --cachedやGit履歴削除の調査では、詳細な方法と影響分析を実施し、ユーザーの判断材料を提供できた
- 5回のイテレーションで問題を段階的に深掘りし、最終的に適切な結論に到達できた

### Problem
- 初回の作業ではの削除のみに焦点を当て、絶対パス情報の削除漏れがあった
- レビューを受けるまで、他の秘匿情報の存在を包括的に調査できていなかった
- 結果として5回のイテレーションが必要となり、作業が分散した

### Try
- 次回からは、秘匿情報削除のタスクでは初回から包括的な調査を実施する
  - 対象となる秘匿情報の種類を事前にリスト化（PC名、ユーザー名、絶対パス、メールアドレス等）
  - 各種類について網羅的に検索を実施
- ドキュメント修正タスクでは、類似の問題が他にも存在しないか横展開して確認する習慣をつける
