# Chrome Web Store公開要件チェックリスト

## 概要

Chrome Web Store への拡張機能公開に必要な要件を整理したチェックリストです。2024年の最新ポリシーに基づいています。

## 必須要件

### 1. プライバシーポリシー関連
- [x] **プライバシーポリシーの作成**: ユーザーデータを取り扱う場合は必須
- [x] **開発者ダッシュボードでの登録**: Chrome Web Store Developer Dashboardの指定フィールドにリンクを設定
- [x] **GitHub.comリンクの使用**: GitHub Pages等でのホスティングは許可されている（調査結果確認済み）
- [x] **内容要件の確認**: データ収集・使用・共有方法を包括的に開示

### 2. プライバシーポリシー内容要件
- [x] **データ収集の説明**: どのようなデータを収集するか明記
- [x] **データ使用目的**: 収集したデータをどのように使用するか説明
- [x] **データ共有**: 第三者とのデータ共有について記載
- [x] **ローカルストレージ**: Chrome Storage APIの使用について説明
- [x] **データを取り扱わない場合**: 個人情報を取り扱わない旨を明記

### 3. セキュリティ要件
- [x] **データ転送の暗号化**: 個人・機密データは最新の暗号化技術で転送
- [x] **セキュアな処理**: ユーザーデータの安全な取り扱い
- [x] **最小権限の原則**: 必要最小限の権限のみリクエスト
- [x] **権限が必要な理由作成**: 各権限の使用理由を詳細に文書化

### 4. 製品情報
- [x] **説明文の作成**: 拡張機能の機能を明確に説明
- [x] **スクリーンショット**: 拡張機能の使用方法を示す画像
- [x] **アイコン**: 16px, 48px, 128px の各サイズのアイコン
- [x] **カテゴリ選択**: 適切なカテゴリの選択

### 5. マニフェスト確認
- [x] **manifest.json の検証**: 正しい形式とバージョン
- [x] **権限の最小化**: 必要最小限の permissions のみ指定
- [x] **アイコンパスの確認**: 各サイズのアイコンファイルが正しく配置

## frog-frame-front 拡張機能固有の確認事項

### データ取り扱い状況
- **DOM操作機能**: ページ内容の読み取りと変更
- **ストレージ機能**: ローカルストレージでのルール保存
- **権限**: activeTab, storage, contextMenus

### プライバシーポリシー要点
1. **データ収集**: 
   - DOM要素の選択情報（一時的、ローカルのみ）
   - ユーザー設定の置換ルール（ローカルストレージ）

2. **データ使用目的**:
   - DOM要素の置換機能の提供
   - ユーザー設定の永続化

3. **データ共有**: 
   - 外部サーバーへのデータ送信なし
   - 全てローカル処理
