UXの改善。
## 右クリックメニューの追加
- 現在`この部分を置換`のメニューがあるので、並列するメニューとして、`この要素を置換`を追加する。

## この要素を置換の挙動

### 実現したいこと
- 表示中ページの任意のテキストを選択し、右クリックメニューから`この要素を置換`を選択すると、選択したテキストを置換するためのメニューが表示され、クリックするとポップアップの`置換前`に反映される

### ユーザーストーリー

- 下記のhtmlで表されるページがあるとする
```html
<center>
<font size="+2">
<i>私たちは以下の原則に従う:</i>

</font>

<p><font size="+2">
顧客満足を最優先し、<br>価値のあるソフトウェアを早く継続的に提供します。
</font></p>

<!-- ここに他の要素が続く -->

</center>
```



- このページの `顧客満足を最優先` をドラッグして選択し、右クリックメニューから `この要素を置換` をクリックする。
- するとポップアップが開き、その `置換前` フォームに `顧客満足を最優先し、<br>価値のあるソフトウェアを早く継続的に提供します。` が反映される
- つまり、ドラッグした部分を含む要素全体が `置換前` に反映される。

# 心配事
<!-- チケットを進めるうえでハードルとなりそうな懸念点を列挙します -->
- 右クリックメニューの実装において、選択したテキストを正確に取得し、ポップアップに反映するためのロジックが複雑になる可能性がある。
- 選択されたテキストを含む最小のDOM要素を特定するためのアルゴリズムが必要になる。
- 選択されたテキストが複数のHTML要素にまたがる場合、どの要素を基準に置換前のテキストを取得するかの判断が難しい。
