# ISSUE-043 PULL REQUEST

## タイトル
コンテキストメニュー作成ロジックをApplication層にリファクタリング

## 概要と理由
コンテキストメニューの作成ロジックをbackground.tsからapplication層に移管するリファクタリングを実施しました。これにより関心の分離を図り、アーキテクチャを改善しました。

背景スクリプト（background.ts）に直接コンテキストメニューの詳細な設定が記述されていましたが、これをApplication層のUseCaseパターンに従って整理することで、保守性と可読性を向上させています。

## 主な変更点
- `ContextMenuSetupUseCase.ts`を新規作成し、コンテキストメニュー設定ロジックを実装
- `background.ts`からコンテキストメニュー作成の詳細ロジックを削除し、UseCaseを呼び出すように変更
- コンテキストメニューの設定処理をApplication層に移管

## テスト方法
1. 拡張機能をビルドしてブラウザにロード
2. Webページでテキストを選択
3. 右クリックしてコンテキストメニューを表示
4. "frog-frame-front" > "この要素を置換" のメニューが正常に表示されることを確認
5. npm run unused:safe を実行し、未使用コードがないことを確認

## 補足
このリファクタリングにより、背景スクリプトがより軽量になり、責任の分離がしっかりとできました。今後のコンテキストメニュー機能の拡張や修正も、Application層で一元管理できるようになります。
