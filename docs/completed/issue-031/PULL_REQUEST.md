# ISSUE-031: CRLF・空白を無視した文字列置換機能の実装

## タイトル
改行コード・空白を無視した文字列置換機能の実装

## 概要と理由
HTMLのソースコードでは、タグ内に改行コードや空白文字が含まれることがある。例えば、アジャイルマニフェストサイト（https://agilemanifesto.org/iso/ja/manifesto.html）の `<h1>` タグは実際には `<h1>アジャイルソフトウェア開発宣言\n</h1>` のように改行が含まれている。

これまでの実装では文字列の完全一致でのみ置換が可能だったため、ユーザーが `<h1>アジャイルソフトウェア開発宣言</h1>` と入力しても置換できない問題があった。

本プルリクエストでは、正規表現を使わない場合でも改行コードや空白を無視して文字列マッチングができるよう機能を拡張した。

## 主な変更点
- **HtmlContent.ts**: `replaceWith` メソッドを修正し、正規表現を使わない場合に改行・空白を無視したマッチングロジックを実装
- **e2eテスト追加**: `ignore-crlf-replace.spec.ts` を作成し、実際のアジャイルマニフェストサイトでの動作を確認するテストを実装
- **ユニットテスト修正**: `HtmlContent.test.ts` に改行コード無視機能のテストケースを追加
- **その他のテストファイル調整**: 既存テストが新しい動作に適合するよう必要な修正を実施

```bash
cd ~/absolute-path/to/favorite-keyword-link-frog && docker compose exec frontend npm test

```

### 手動テスト
1. `https://agilemanifesto.org/iso/ja/manifesto.html` をブラウザで開く
2. 拡張機能のポップアップを開く
3. 以下を入力：
   - URLパターン: `https://agilemanifesto.org`（自動入力される）
   - 置換前: `<h1>アジャイルソフトウェア開発宣言</h1>`
   - 置換後: `<h2>アジャイルソフトウェア開発宣言</h2>`
   - 正規表現チェックボックス: 空のまま
4. 保存ボタンを押す
5. ページ上の `<h1>` タグが `<h2>` に置換されることを確認

## 補足
- **アーキテクチャ**: CODING_STYLE.mdのオブジェクト指向ルールに従い、クリーンアーキテクチャを維持
- **後方互換性**: 既存の正規表現を使った置換機能には影響なし
- **パフォーマンス**: 改行・空白の正規化処理は正規表現を使わない場合のみ実行されるため、既存機能への性能影響は最小限
- **実装アプローチ**: 文字列の正規化により空白・改行を除去してマッチング後、元のHTML内の対応する範囲を特定して置換する方式を採用
